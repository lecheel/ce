codeium-editor

Mini vim-buffer TUI editor with AI completions, split layouts, Brief mode, and dynamic configuration

Rust 2021 Ratatui + Crossterm Codeium + Llama.cpp LLM Vim-like Modal Sequences Brief Mode (Always-Insert) Tokio Async

🏗 Module Architecture

main.rs — CLI & Event Loop
  • clap CLI: path args, auth subcommand, status subcommand
  • tokio mpsc event bus: AppMessage::Input / CompletionResponse / Tick
  • Codeium server spawn + API key gated by codeium_enabled config flag — skipped entirely when false
  • Request Version IDs protect against async completions overwriting newer state
  • Crossterm KeyPress-only filtering (Release/Repeat events dropped at source)
  • Local completion async task reads pre-cached buffer_words + vocab_words + active line delta
ed/ — Editor Core
  • buffer.rs — Rope-backed Buffer, UndoSnapshot, file I/O, rustfmt on .rs save, active bookmarks, and Git change states
  • window.rs — Independent viewports, desired_col (curswant), LayoutNode binary tree, WindowPosition, get_selection_range() selection calculations
  • editor.rs — Request versioning, history I/O, buffer word cache refresh, QuitPrompt, Brief/Vim mode state, and get_word_under_cursor() helper routines
  • movement.rs — hjkl, word, line, page (viewport-aware) movement free-functions
  • editing.rs — Auto-indent Enter, soft-tab, indent/outdent, paste helpers, desired_col sync, and fine-grained AST updates
  • repeat.rs — Vim dot repeat (.) engine leveraging structured RepeatableAction payloads and automated scale multipliers
  • gutter.rs — Absolute/relative line numbers, colored Git diff gutter signs, visual bookmarks
  • syntax.rs — Tree-sitter integration, scope detection, highlighting, and quote/brace/parameter text objects
  • ext.rs — EditorExt trait (plugin hook stubs)
comp/ — Completion
  • state.rs — CompletionMachine: ghost-text, cycling, 400 ms throttle, request IDs
  • provider.rs — CompletionProvider async trait (backend-agnostic)
  • Contextual guards: blocks mid-word, next-to-), and before-) positions
  • find_prefix_overlap() prevents character duplication on accept
ai/ — AI & LLM Backends
  • codeium/ — Cloud gRPC-JSON completions, LSP helper thread, browser login auth, and Windsurf/VSCode fallback cache scrapers.
  • llama/llm.rs — Custom local llama.cpp integration over raw TCP stream. Communicates using OpenAI-compatible /v1/chat/completions payload mappings to process model-native templates.
  • Powers both interactive split-screen LLM chat sessions (:llm) and the automated conventional git commit generator (:gc / status screen c).
render/ — UI
  • buffer_view.rs — Render active windows with styled line display utilizing gutter.rs for dynamic sidebar layout, visual selection overlay, and on-screen query highlights
  • tabs.rs — Multi-buffer tab bar (hidden when single buffer)
  • statusbar.rs — Mode pill, filename with Fish-style contraction, [+], row:col, lang, Struct::fn() scope, Codeium spinner/OK/OFF
  • command_line.rs — : command + top-5 candidate preview OR status message. Layout: very bottom line.
  • helpers.rs — CJK-aware char width, digit count, language display names, and Fish-style path contracting algorithm
keybind/ — Keybindings
  • actions.rs — EditorAction enum (pure data, no UI deps)
  • bindings.rs — Action enum, format_key, resolve_sequence, resolve_single_key, Brief Alt-key layer, and bracket normalizer
  • Leader Key dynamic multi-key sequences with mapleader override support
  • Unified Namespaced configuration: keybindings.normal, keybindings.insert, keybindings.visual, keybindings.global
  • Text Objects: diw, ci", di(, ci{, etc.
config/ — Configuration
  • app_config.rs — Config struct with load/save to ~/.config/ce/config.json
  • Fields: api_key, api_url, portal_url, max_tokens, codeium_enabled, popup_enabled, init_mode, keybindings, leader, search_wrap_enabled
  • Auto-discovers nvim/codeium key on load if api_key is absent
repl/ — Command REPL
  • command.rs — :q :q! :w :wq :x :e :new :ls :bn :bp :bd :b N :config :vocab :vim :brief :sp :vs :on :gs :stash
  • Tab path completion with ./ and ../ relative prefix preservation
  • Command name prefix completion via complete_command(), integrated with persistent history search
popup/ — Popup System
  • mod.rs — PopupState, PopupItem, PopupKind (Completion/CommandPalette/Hover/Custom)
  • Centered floating config dashboard — runtime Serde JSON bool reflection + Space toggle
  • Bottom-right Which-Key hint with compact modifier notation (s+, c+, a+), synced with 30-char min width and unified theme colors
git/ — Git State Machinery
  • status.rs — Git status screen parser. Retains raw 2-character columns (prevents whitespace-trimming bugs) to map Stage Changes, Unstage Changes, Untracked, Branches, and Stashes.
  • PendingGitAction — Safety intercept dialog state-machine. Generates warning prompts on-screen for checkout and pop actions.
  • Automatic file buffer reloading, active syntax parsing, gutter diff updates, and window cursor clamping upon successful branch checkouts.

🐛 Known Issues & Fixes

Fixed — Git Status parsing column shifts
In the initial status screen, using let trimmed = raw_line.trim(); removed leading spaces from unstaged modified files (e.g. M main.rs). This shifted the second-column M flag to index 0, causing the parser to incorrectly classify unstaged changes as staged. Fixed by preserving raw lines during row index parsing.
Fixed — Visual mode selection movement keys blocked
In Editor::handle_key(), if the active mode was not Normal, we returned early. This blocked navigation keys like h/j/k/l from being processed, making Visual selection completely static. Fixed by allowing unmatched visual inputs to fall through to standard multi-key and movement resolvers.
Fixed — Esc key unresponsive in Visual modes
In bindings.rs, Action::ExitMode lacked match arms for Mode::Visual and Mode::VisualLine, so pressing Esc in Visual modes did nothing. Fixed by adding a branch to transition back to Normal mode and clear the selection anchor.
Fixed — Custom Leader immediately executes unique actions (AutoAction bug)
In keybind/bindings.rs, if only one keybinding matched the pressed leader key prefix (e.g., , q q), resolve_sequence() returned AutoAction, instantly firing the command on the first keypress. Fixed by returning ResolveResult::Pending on all partial prefix matches, allowing the user to complete their custom sequences.
Fixed — Double mutable borrow compiler conflict on Undo
In keybind/bindings.rs, attempting to simultaneously mutably borrow the buffer (to pop undo states) and the active window (to restore cursor coordinates) caused compiler errors. Fixed by leveraging the editor's disjoint borrow helper active_window_and_buf_mut().
Fixed — Codeium ghost text never shows after first request
In comp/state.rs, set_completions() and cancel_pending() previously did not clear last_checked_pos. After the first request fired for position (row, col), last_checked_pos was permanently set, causing maybe_request() to return None for all future ticks at the same cursor position. Fixed by ensuring position state is correctly cleared upon receiving a response or cancellation.
Fixed — Inline [1/8] overwriting ghost text
The completion indicator [idx/total] rendered inline on the cursor line was overwriting the end of ghost text. It has been removed from buffer_view.rs. Completion counts and candidates are now exclusively shown in the bottom command line area, leaving ghost text clean and uninterrupted.
Fixed — Cursor column resets to 0 over empty lines
Moving vertically (j/k/PageUp/PageDown) would clamp win.col to the line length (0 for empty lines), losing the original horizontal position. Fixed by adding a desired_col field (Vim's "curswant") to Window, which remembers the target column and restores it when returning to a longer line.
Fixed — Syntax highlighting not applying
Buffer::new() and open_file() never called parse_syntax(), so the syntax tree was always None on startup. Additionally, the catch-all arm Action::InsertChar(_) | Backspace | ... => parse_syntax() in execute_action was unreachable because those variants were already matched by earlier arms. Fixed by adding parse_syntax() calls on buffer load and moving the parse call to the end of execute_action.
Fixed — Command Line and Status Bar swapped
The command/message line and the status bar positions were inverted compared to standard Vim layout. The command/message line now correctly sits at the very bottom of the screen, with the status bar immediately above it.
Fixed — Which-Key and Config popups not rendering
The draw_config_popup and draw_key_sequence_hint functions existed but were never called from the main render::draw() function. They have been added to the render pipeline.

✨ Implemented Features

⌨️ Vim-like Modal Editing

Normal (hjkl, dd, yy, u, x, gg, G), Insert (direct typing, soft-tabs, auto-indent on Enter), Visual/Visual-Line (styled overlays with hardware cursor tracking), Command (: prefix), Brief (always-insert). Transitions: i/I/a/A/o/O enter insert, gi enters Brief, v/V enter Visual selections, Esc returns to Normal with cursor-left adjustment.

Complete

🚀 Brief Mode (Always-Insert)

An alternative editing mode where all printable keys insert text, and commands are accessed via Alt shortcuts (e.g., Alt+s save, Alt+d delete line, Alt+u undo, Alt+x exit to Vim). F9 opens the command line. Esc clears ghost text but stays in Brief. Set "init_mode": "brief" in config to start in Brief mode. Switch modes with :vim and :brief.

Complete

🪟 Split Window Layouts

Split the active window horizontally (:sp / Ctrl+w s) or vertically (:vs / Ctrl+w v). Navigate with Ctrl+w h/j/k/l or Ctrl+w w. Close with Ctrl+w q or :q. :on closes all other windows. Dividers rendered between panes.

Complete

📄 Multi-Buffer Editing

Open multiple files as separate buffers with tab bar (hidden for single buffer, [+] modified flag). Commands: :e <path>, :bn/:bp, :bd, :ls, :b N. refresh_buffer_words() is triggered on every open/switch/close for accurate local autocomplete.

Complete

🤖 AI Ghost-Text Completions (Codeium)

Completions from local Codeium LSP via gRPC-JSON gateway. Inline ghost text with prefix-overlap dedup and suffix-overlap trimming (no inline [1/8] clutter). Tab or Right-Arrow accepts; Ctrl+N/P or Up/Down (when ghost active) cycles candidates. Contextual guards block mid-word, )-adjacent triggers. Works reliably in both Insert and Brief modes.

Complete

🤖 Local LLM & Conventional Commits

Generates semantic conventional git commits from staged diffs using a local llama.cpp server (via /v1/chat/completions). Features a split-pane interactive chat session (:llm) allowing real-time prompt querying. Supported by robust raw-TCP network sockets and background async pollers.

Complete

🌳 Tree-sitter Syntax & Text Objects

Incremental Tree-sitter parsing for Rust, Python, JS/TS. Syntax-aware highlighting with correct "most specific node wins" logic. Text objects: diw/ciw, di"/ci", di(/ci(, di{/ci{, di[/ci], dif/cif. Contextual scope (e.g., Struct::fn or Enum) displayed dynamically on the status bar right.

Complete

🔍 Vim-style Search & Highlighting

Forward search with / and navigation via n/N with Vim-compliant wrap-around behavior (controlled by search_wrap_enabled in config). Pressing * grabs the word under/after the cursor and jumps directly to its next occurrence. Matches are highlighted on-screen with high-contrast yellow background.

Complete

🔑 Leader Key & Mapleader Overrides

Vim-style sequences like space w to save, space q to quit. By defining "leader": "," in config.json, the editor dynamically overrides the default system leader "space" mappings to use your custom leader, freeing up the spacebar completely.

Complete

🐙 Interactive Git Workspace Safety

Dedicated *git-status* buffer. Renders staged changes (no-symbol green), unstaged changes (bracketed yellow), untracked files (red), branch history (active starred), and the last 10 stashes. Protected by a prompt-level safety flow (y/n) for switches and pops.

Complete

🐙 Git Gutter Diff & Gutter Display

A unified gutter rendering system with support for absolute or hybrid relative line numbers, colored Git diff columns (green ┃ for additions, yellow ┃ for edits, red ┵ for removals), and visual bookmarks (represented by purple stars ★), all togglable dynamically with custom shortcuts.

Complete

↩️ Vim Dot Repeat (.)

A structural, state-preserving repeat mechanism. Using a RepeatAction enum, the editor tracks and replicates standalone normal mode modifications (e.g., dd, x), insert typing sessions (including multi-line edits), and complex text object changes (like ciw).

Complete

📋 Which-Key Popup

Bottom-right floating popup shows downstream keys available for the current pending sequence. Compact modifier notation (shift+ → s+, ctrl+ → c+, alt+ → a+). Scans both default actions and user config keybindings. Toggled via space t or :config → popup_enabled.

Complete

🛠️ Local Autocomplete Fallback

When Codeium is disabled, completions come from three merged sources: pre-cached vocab words (length ≥ 4, loaded from wordlist.txt), pre-cached buffer words (length ≥ 6, lazy-indexed at load/save/switch), and an incremental delta scan of the current active line to capture newly typed identifiers. All run non-blocking via tokio::spawn.

Complete

↩️ Register Copy-Paste & Undo

Yank current line via yy (or as side-effect of dd) into a unified clipboard register. Paste below current line via space p v. Line-yanked text (ends with \n) uses paste_line_below(); char-yanked text uses inline paste_text(). Per-buffer undo stack: 500 rope snapshots, reverted via u.

Complete

💾 File I/O with Auto-Format

Rope-backed open/save. :w writes to disk; .rs files automatically run rustfmt then reload, restoring cursor within bounds. :w <path> sets filename then saves. Rope always maintains trailing newline. Tab completion for :e/:w/:wq with ./ and ../ prefix preservation.

Complete

⚙️ Reflection Config Dashboard

:config opens a centered overlay. Uses runtime serde_json::to_value(&config) to discover all boolean fields dynamically — no hardcoded list. Space toggles, Esc closes. Toggle is saved to disk immediately. Key names are humanized (codeium_enabled → Codeium Enabled).

Complete

📝 Persistent Command History

Command mode input is logged to ~/.config/ce/history.txt (max 500 entries, dedup consecutive). Up/Down arrows navigate backward/forward; on reaching the bottom, the pre-navigation draft is restored from cmd_temp_input. History is loaded at startup.

Complete

🧭 Smart Cursor & Viewport Movement

desired_col (curswant) remembers your horizontal position when moving vertically over short or empty lines, snapping back when returning to a long line. PageUp/PageDown adapt to the current window's viewport height rather than a hardcoded jump value.

Complete

🧹 Dirty Buffer Quit Prompt

Attempting to quit (:q, space q) with unsaved changes triggers an interactive overlay: y saves then quits/closes, n discards and quits/closes, c or Esc cancels. Prompt is rendered as a red-bold status message. :q! and space Q force-quit without prompting.

Complete

🦀 Technology Stack

Core Runtime

Tokio (full) — async runtime, mpsc channels, interval timers, tokio::spawn.
Crossterm 0.29 — raw mode, press-only key events, alternate screen.
Ratatui 0.29 — declarative TUI: Paragraph, Layout, Span, Line, Clear, Block.

Text Buffer & Syntax

Ropey 1 — rope data structure for O(log n) inserts/deletes. Each Buffer owns a Rope; line_to_char() gives O(1) line→char index. Trailing newline invariant is always maintained.
Tree-sitter — incremental parsing, syntax highlighting, and text object/scope queries for Rust, Python, JS/TS.

Networking

reqwest 0.12 (rustls-tls, json, blocking) — async HTTP client for Codeium LSP + cloud; blocking client for binary download.
sha2 — binary integrity SHA-256 verification on download.
flate2 — gzip decompression of downloaded LSP binary.

CLI & Config

clap 4 (derive) — path args + auth/status subcommands.
serde / serde_json — JSON reflection for config toggle dashboard + API types.
dirs — cross-platform config/cache/home/data paths.
uuid — session ID and auth state generation.
anyhow — error handling throughout.

🐙 Git Status Buffer Shortcuts

These shortcuts are active strictly when focused on the *git-status* panel in Normal mode:
s — Toggle stage / unstage status for the file under the cursor (unstages if staged, stages if unstaged/untracked)
c — Auto-stage all tracked modifications and open an LLM commit message generation task
z — Pre-populate Command mode with :stash to prompt-fill a custom stash description
Enter — contextual action based on row: open file in active window / switch branch / pop stash
y / n — confirm or cancel interactive prompts (e.g. branch checkout safety locks or destructive stash pops)
q / Esc — safely exit the status panel, restoring whichever mode you had active previously (Vim or Brief)

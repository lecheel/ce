codeium-editor

Mini vim-buffer TUI editor with AI completions, split layouts, Brief mode, and dynamic configuration

Rust 2021 Ratatui + Crossterm Codeium + Llama.cpp LLM Vim-like Modal Sequences Brief Mode (Always-Insert) Tokio Async

🏗 Module Architecture

main.rs — CLI & Event Loop
  • clap CLI: path args, auth subcommand, status subcommand
  • tokio mpsc event bus: AppMessage::Input / CompletionResponse / Tick
  • Codeium server spawn + API key gated by codeium_enabled config flag — skipped entirely when false
  • Request Version IDs protect against async completions overwriting newer state
  • Crossterm KeyPress-only filtering (Release/Repeat events dropped at source)
  • Local completion async task reads pre-cached buffer_words + vocab_words + active line delta
ed/ — Editor Core
  • buffer.rs — Rope-backed Buffer, UndoSnapshot, file I/O, rustfmt on .rs save, active bookmarks, and Git change states
  • window.rs — Independent viewports, desired_col (curswant), LayoutNode binary tree, WindowPosition, get_selection_range() selection calculations
  • editor.rs — Request versioning, history I/O, buffer word cache refresh, QuitPrompt, Brief/Vim mode state, Ctrl+R register mode intercept, and precise undo boundary tracking
  • movement.rs — hjkl, word, line, page (viewport-aware) movement free-functions
  • editing.rs — Auto-indent Enter, soft-tab, indent/outdent, paste helpers, desired_col sync, multi-byte safe newline handling, and fine-grained AST updates
  • repeat.rs — Vim dot repeat (.) engine leveraging structured RepeatableAction payloads, intelligent insert session grouping, and automated scale multipliers
  • gutter.rs — Absolute/relative line numbers, colored Git diff gutter signs, visual bookmarks
  • syntax.rs — Tree-sitter integration, scope detection, Catppuccin Mocha highlighting, text objects, and AST-driven bracket matching (%)
  • tag.rs — Ctag/gentag integration manager for definition lookups, persistent caching, and tag stack back-navigation
  • handle/subst.rs — Interactive substitution (:%s/foo/bar/gc) with prefix search, character boundary safety, and multi-candidate confirmations
  • ext.rs — EditorExt trait (plugin hook stubs)
comp/ — Completion
  • state.rs — CompletionMachine: ghost-text, cycling, 400 ms throttle, request IDs
  • provider.rs — CompletionProvider async trait (backend-agnostic)
  • Contextual guards: blocks mid-word, next-to-), before-), and minimum 4-char prefix length
  • Stale response rejection: on_edit() invalidates in-flight requests; on_response() re-validates cursor position
  • find_prefix_overlap() prevents character duplication on accept
ai/ — AI & LLM Backends
  • codeium/ — Cloud gRPC-JSON completions, LSP helper thread, browser login auth, and Windsurf/VSCode fallback cache scrapers.
  • llama/llm.rs — Custom local llama.cpp integration over raw TCP stream. Communicates using OpenAI-compatible /v1/chat/completions payload mappings to process model-native templates.
  • Powers both interactive split-screen LLM chat sessions (:llm) and the automated conventional git commit generator (:gc / status screen c).
render/ — UI
  • buffer_view.rs — Render active windows with styled line display utilizing gutter.rs for dynamic sidebar layout, visual selection overlay, indent guides, and on-screen query highlights
  • tabs.rs — Multi-buffer tab bar (hidden when single buffer)
  • statusbar.rs — Mode pill, filename with Fish-style contraction, [+], row:col, lang, Struct::fn() scope, Codeium spinner/OK/OFF
  • command_line.rs — : command + top-5 candidate preview OR status message. Layout: very bottom line. Supports visual ^ rendering for Ctrl+R register insertions.
  • helpers.rs — CJK-aware char width, digit count, language display names, and Fish-style path contracting algorithm
keybind/ — Keybindings
  • bindings.rs — Action enum with strum derive macros for auto-generated snake_case names and declarative aliases. Format_key, resolve_sequence, resolve_single_key.
  • palette_defaults.rs — Compile-time default descriptions for the command palette.
  • desc_override.rs — Load user overrides from ~/.config/ce/desc.json via :gen_desc.
  • Leader Key dynamic multi-key sequences with mapleader override support
  • Unified Namespaced configuration: keybindings.normal, keybindings.insert, keybindings.visual, keybindings.global
  • Text Objects: diw, ci", di(, ci{, etc.
config/ — Configuration
  • app_config.rs — Config struct with load/save to ~/.config/ce/config.json
  • Fields: api_key, api_url, portal_url, max_tokens, codeium_enabled, popup_enabled, init_mode, keybindings, leader, search_wrap_enabled
  • Auto-discovers nvim/codeium key on load if api_key is absent
repl/ — Command REPL
  • command.rs — :q :q! :w :wq :x :e :new :ls :bn :bp :bd :b N :config :vocab :vim :brief :sp :vs :on :gs :stash :tig :gen_desc :mru :mru! :command_palette :scankey :guide :checkhealth :diffthis :gd :sort :%s :tag :retag
  • Tab path completion with ./ and ../ relative prefix preservation
  • Command name prefix completion via complete_command(), integrated with persistent history search
  • Visual range execution support (e.g., :'<,'>s/foo/bar/g, :'<,'>sort)
popup/ — Popup System
  • mod.rs — PopupState, PopupItem, PopupKind (Completion/CommandPalette/Hover/Custom)
  • Centered floating config dashboard — runtime Serde JSON bool reflection + Space toggle
  • Bottom-right Which-Key hint with compact modifier notation (s+, c+, a+), synced with 30-char min width and unified theme colors
  • Command Palette with desc.json user overrides, fuzzy matching, and :gen_desc auto-generation
  • Advanced Marks navigation popup with fuzzy search anchors for instant filtering/jumping
  • Tag Candidates popup for disambiguating multiple matches on C-]
git/ — Git State Machinery
  • status.rs — Git status screen parser. Retains raw 2-character columns (prevents whitespace-trimming bugs) to map Stage Changes, Unstage Changes, Untracked, Branches, and Stashes.
  • log.rs — Configurable tig-like viewer. Supports limits (:tig defaults to 10, :tig 0 for all, :tig 20 for N).
  • PendingGitAction — Safety intercept dialog state-machine. Generates warning prompts on-screen for checkout and pop actions.
  • Automatic file buffer reloading, active syntax parsing, gutter diff updates, and window cursor clamping upon successful branch checkouts.

🐛 Known Issues & Fixes

Fixed — 1-char ghost text / Stale Async Completion Bug
Pressing backspace quickly after typing could leave 1-character ghost text because stale in-flight responses were accepted. Fixed by incrementing request_id on every edit (invalidating old requests) and re-checking the 4-char minimum prefix context inside on_response() before rendering.
Fixed — Brief Mode Undo Grouping Gaps
In Brief mode, undoing a line deletion would also undo all previous typing. Fixed by implementing an undo boundary state machine: continuous typing (InsertChar, Backspace, Newline) merges into a single undo block, while discrete commands (DeleteCurrentLine, Paste, ToggleComment) push a new snapshot. The recording engine properly flushes the insert buffer when interrupted by destructive actions.
Fixed — Ctrl+C / Ctrl+X cancelling Brief selection instead of acting on it
Pressing copy/cut while text was selected via Alt+L cleared the selection instead of copying/cutting it. Fixed by adding DeleteCurrentLine and YankCurrentLine to the selection preservation list, and updating their execution logic to detect the visual anchor and act only on the highlighted text.
Fixed — Match Bracket (%) breaking Brief selection
Pressing % to jump to a matching brace cancelled the visual anchor. Fixed by adding Action::MatchBracket to the Brief selection preservation list, allowing the cursor to jump to the matching brace while keeping the initial anchor, effectively selecting the block contents.
Fixed — Git Status parsing column shifts
In the initial status screen, using let trimmed = raw_line.trim(); removed leading spaces from unstaged modified files (e.g. M main.rs). This shifted the second-column M flag to index 0, causing the parser to incorrectly classify unstaged changes as staged. Fixed by preserving raw lines during row index parsing.
Fixed — Visual mode selection movement keys blocked
In Editor::handle_key(), if the active mode was not Normal, we returned early. This blocked navigation keys like h/j/k/l from being processed, making Visual selection completely static. Fixed by allowing unmatched visual inputs to fall through to standard multi-key and movement resolvers.
Fixed — Custom Leader immediately executes unique actions (AutoAction bug)
In keybind/bindings.rs, if only one keybinding matched the pressed leader key prefix (e.g., , q q), resolve_sequence() returned AutoAction, instantly firing the command on the first keypress. Fixed by returning ResolveResult::Pending on all partial prefix matches, allowing the user to complete their custom sequences.
Fixed — Cursor column resets to 0 over empty lines
Moving vertically (j/k/PageUp/PageDown) would clamp win.col to the line length (0 for empty lines), losing the original horizontal position. Fixed by adding a desired_col field (Vim's "curswant") to Window, which remembers the target column and restores it when returning to a longer line.
Fixed — Command Line and Status Bar swapped
The command/message line and the status bar positions were inverted compared to standard Vim layout. The command/message line now correctly sits at the very bottom of the screen, with the status bar immediately above it.
Fixed — Multi-byte character (Emoji) corruption on Enter
Pressing Enter in the middle or end of a line containing emojis (like 🔴) corrupted the buffer, and pressing Enter at the end of a line merged the next line into it. Fixed by using character-based indexing (.chars()) instead of byte-based slicing during line splitting, and explicitly preserving the original trailing newline character during Rope manipulation.
Fixed — Visual selection dismissed when entering Command mode
Pressing : from Visual mode to run :'<,'>s/foo/bar/g immediately cleared the visual highlight. Fixed by saving and restoring the visual_anchor across the mode transition, and updating the renderer to keep displaying the visual highlight while in Command mode if an anchor exists.

✨ Implemented Features

⌨️ Vim-like Modal Editing

Normal (hjkl, dd, yy, u, x, gg, G), Insert (direct typing, soft-tabs, auto-indent on Enter), Visual/Visual-Line/Visual-Block (styled overlays with hardware cursor tracking), Command (: prefix), Brief (always-insert). Transitions: i/I/a/A/o/O enter insert, gi enters Brief, v/V/Ctrl+v enter Visual selections, Esc returns to Normal with cursor-left adjustment. Unified bookmark system: m sets named marks (a-z), ` opens a navigable marks popup. % jumps to matching bracket using Tree-sitter AST.

Complete

📐 Visual Block Mode & Column Editing

Ctrl+v enters rectangular visual selection. I inserts at the start of the block on every selected line (column insert). A appends after the block on every selected line. After typing, Esc replicates the typed text to all rows of the block. Supports yank (y), delete (d/x), and change (c) for rectangular regions.

Complete

🚀 Brief Mode (Always-Insert)

An alternative editing mode where all printable keys insert text, and commands are accessed via Alt and Ctrl shortcuts. Features inline visual selection via Alt+L (extend with Shift+Nav), Ctrl+C/Ctrl+X/Ctrl+V for copy/cut/paste (acts on selection if active, else NOP for C/X, normal paste for V), Alt+Y for yanking the current word (or active selection), Alt+M for auto-incrementing named bookmarks, and Alt+J for mark jumps. Alt+G or F9 opens the command line. Esc clears ghost text but stays in Brief. Set "init_mode": "brief" in config to start in Brief mode. Switch modes with :vim and :brief. Intelligent undo grouping separates continuous typing from discrete destructive commands.

Complete

🪟 Split Window Layouts

Split the active window horizontally (:sp / Ctrl+w s) or vertically (:vs / Ctrl+w v). Navigate with Ctrl+w h/j/k/l or Ctrl+w w. Close with Ctrl+w q or :q. :on closes all other windows. Dividers rendered between panes.

Complete

📄 Multi-Buffer Editing & MRU

Open multiple files as separate buffers with tab bar (hidden for single buffer, [+] modified flag). Commands: :e <path>, :bn/:bp, :bd, :ls, :b N. :mru opens a fuzzy-searchable recent files popup (defaults to repo-scoped files; :mru! or :mruall shows global history). refresh_buffer_words() is triggered on every open/switch/close for accurate local autocomplete.

Complete

🤖 AI Ghost-Text Completions (Codeium)

Completions from local Codeium LSP via gRPC-JSON gateway. Inline ghost text with prefix-overlap dedup and suffix-overlap trimming (no inline [1/8] clutter). Tab or Right-Arrow accepts; Ctrl+N/P or Up/Down (when ghost active) cycles candidates. Contextual guards block mid-word, )-adjacent triggers, and enforce a 4-character minimum prefix. Stale requests are strictly discarded by ID. Works reliably in both Insert and Brief modes.

Complete

🤖 Local LLM & Conventional Commits

Generates semantic conventional git commits from staged diffs using a local llama.cpp server (via /v1/chat/completions). Features a split-pane interactive chat session (:llm) allowing real-time prompt querying. Supported by robust raw-TCP network sockets and background async pollers.

Complete

🌳 Tree-sitter Syntax & Text Objects

Incremental Tree-sitter parsing for Rust, Python, JS/TS. Syntax-aware highlighting (Catppuccin Mocha palette) with correct "most specific node wins" logic. Text objects: diw/ciw, di"/ci", di(/ci(, di{/ci{, di[/ci], dif/cif. AST-driven bracket matching via % (skips brackets in strings/comments). Contextual scope (e.g., Struct::fn or Enum) displayed dynamically on the status bar right.

Complete

🔍 Vim-style Search & Substitution

Forward search with / and navigation via n/N with Vim-compliant wrap-around behavior (controlled by search_wrap_enabled in config). Pressing * grabs the word under/after the cursor and jumps directly to its next occurrence. Matches are highlighted on-screen with high-contrast yellow background. :noh clears highlights. Interactive substitution via :%s/old/new/gc (with y/n/a/q prompt), visual range support :'<,'>s/old/new/g, and single-line :s/old/new/. Character-boundary safe (handles multi-byte chars/emojis correctly).

Complete

🏷️ Tag Navigation (Ctags)

Jump to symbol definitions using C-] (tag under cursor) or :tag <name>. If multiple matches exist, a bottom-up popup lets you select the exact target. C-t jumps back through the tag stack. :retag regenerates the tags file using gentag or ctags -R. Tag files are cached with stale-modification-time checks for performance.

Complete

🔢 Line Sorting

Sort lines via :sort (whole file) or :'<,'>sort (visual selection). Supports reverse (:sort! or :sort r), unique (:sort u), and case-insensitive (:sort i) flags. Correctly replaces the exact line range in the Rope without duplicating trailing newlines.

Complete

🔖 Unified Marks & Advanced Bookmarks

Unified mark storage bridging Vim and Brief. m (Vim) and Alt+M (Brief) automatically assign the next available letter (a-z) and toggle off if the line is already marked. ` (Vim) and Alt+J (Brief) open a searchable marks popup listing all marks across open buffers. Type to fuzzy-filter by filename or mark character, acting as a search anchor to jump instantly without scrolling. Visual gutter icons (★) sync instantly with mark state.

Complete

🔑 Leader Key & Mapleader Overrides

Vim-style sequences like space w to save, space q to quit. By defining "leader": "," in config.json, the editor dynamically overrides the default system leader "space" mappings to use your custom leader, freeing up the spacebar completely.

Complete

🐙 Interactive Git Workspace & Log

Dedicated *git-status* buffer. Renders staged changes (no-symbol green), unstaged changes (bracketed yellow), untracked files (red), branch history (active starred), and the last 10 stashes. Protected by a prompt-level safety flow (y/n) for switches and pops. :tig opens a tig-like log viewer (defaults to 10 commits, :tig 0 shows all, :tig 20 limits to N).

Complete

🐙 Git Gutter Diff & Gutter Display

A unified gutter rendering system with support for absolute or hybrid relative line numbers, colored Git diff columns (green ┃ for additions, yellow ┃ for edits, red ┵ for removals), visual bookmarks (represented by purple stars ★), and indentation guidelines (│), all togglable dynamically with custom shortcuts.

Complete

↩️ Vim Dot Repeat (.)

A structural, state-preserving repeat mechanism. Using a RepeatAction enum, the editor tracks and replicates standalone normal mode modifications (e.g., dd, x), insert typing sessions (including multi-line edits), and complex text object changes (like ciw). Properly flushes insert buffers when interrupted by destructive actions.

Complete

📋 Command Palette & Overridable Descriptions

VSCode-style command palette (F2 or Ctrl+Alt+P) with fuzzy search. Action names are auto-derived from strum snake_case. Compile-time descriptions can be fully overridden via ~/.config/ce/desc.json. Use :gen_desc to auto-generate a fully populated default file to edit.

Complete

📝 Toggle Commenter

Language-aware line commenting (gcc in Normal, Ctrl+/ in Insert/Brief). Automatically detects comment prefixes (//, #, --, ;;, etc.) and preserves leading indentation. Automatically advances the cursor to the next line for rapid block commenting.

Complete

🛠️ Local Autocomplete Fallback

When Codeium is disabled, completions come from three merged sources: pre-cached vocab words (length ≥ 4, loaded from wordlist.txt), pre-cached buffer words (length ≥ 6, lazy-indexed at load/save/switch), and an incremental delta scan of the current active line to capture newly typed identifiers. All run non-blocking via tokio::spawn.

Complete

↩️ Register Copy-Paste & Undo

Yank current line via yy (or as side-effect of dd) into a unified clipboard register. Paste below current line via p. Line-yanked text (ends with \n) uses paste_line_below(); char-yanked text uses inline paste_text(). Per-buffer undo stack with intelligent boundary tracking. Ctrl+R in command line inserts register specials (% filename, Ctrl+W word, Ctrl+L line number).

Complete

💾 File I/O with Auto-Format

Rope-backed open/save. :w writes to disk; .rs files automatically run rustfmt then reload, restoring cursor within bounds. :w <path> sets filename then saves. Rope always maintains trailing newline. Tab completion for :e/:w/:wq with ./ and ../ prefix preservation.

Complete

⚙️ Reflection Config Dashboard

:config opens a centered overlay. Uses runtime serde_json::to_value(&config) to discover all boolean fields dynamically — no hardcoded list. Space toggles, Esc closes. Toggle is saved to disk immediately. Key names are humanized (codeium_enabled → Codeium Enabled).

Complete

📝 Persistent & Separated History

Command mode input (:) is logged to ~/.config/ce/history.txt. Search mode input (/) is logged separately to ~/.config/ce/search_history.txt. Up/Down arrows navigate backward/forward scoped to the active mode. History is loaded at startup.

Complete

🧭 Smart Cursor & Viewport Movement

desired_col (curswant) remembers your horizontal position when moving vertically over short or empty lines, snapping back when returning to a long line. PageUp/PageDown adapt to the current window's viewport height rather than a hardcoded jump value.

Complete

🧹 Dirty Buffer Quit Prompt

Attempting to quit (:q, space q) with unsaved changes triggers an interactive overlay: y saves then quits/closes, n discards and quits/closes, c or Esc cancels. Prompt is rendered as a red-bold status message. :q! and space Q force-quit without prompting.

Complete

🦀 Technology Stack

Core Runtime

Tokio (full) — async runtime, mpsc channels, interval timers, tokio::spawn.
Crossterm 0.29 — raw mode, press-only key events, alternate screen.
Ratatui 0.29 — declarative TUI: Paragraph, Layout, Span, Line, Clear, Block.

Text Buffer & Syntax

Ropey 1 — rope data structure for O(log n) inserts/deletes. Each Buffer owns a Rope; line_to_char() gives O(1) line→char index. Trailing newline invariant is always maintained.
Tree-sitter — incremental parsing, syntax highlighting, and text object/scope queries for Rust, Python, JS/TS.

Networking

reqwest 0.12 (rustls-tls, json, blocking) — async HTTP client for Codeium LSP + cloud; blocking client for binary download.
sha2 — binary integrity SHA-256 verification on download.
flate2 — gzip decompression of downloaded LSP binary.

CLI & Config

clap 4 (derive) — path args + auth/status subcommands.
serde / serde_json — JSON reflection for config toggle dashboard + API types.
strum — Enum string conversions, auto-snake_case, and declarative alias definitions for Actions.
dirs — cross-platform config/cache/home/data paths.
uuid — session ID and auth state generation.
anyhow — error handling throughout.

🐙 Git Status Buffer Shortcuts

These shortcuts are active strictly when focused on the *git-status* panel in Normal mode:
s — Toggle stage / unstage status for the file under the cursor (unstages if staged, stages if unstaged/untracked)
c — Auto-stage all tracked modifications and open an LLM commit message generation task
z — Pre-populate Command mode with :stash to prompt-fill a custom stash description
Enter — contextual action based on row: open file in active window / switch branch / pop stash
y / n — confirm or cancel interactive prompts (e.g. branch checkout safety locks or destructive stash pops)
q / Esc — safely exit the status panel, restoring whichever mode you had active previously (Vim or Brief)
